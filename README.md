# 共享单车停放热点预测与调度优化系统

本项目是一个面向本科毕业设计、论文截图和本地答辩演示的 Streamlit Web 系统。系统以山东公共数据开放平台梁山县哈啰共享单车订单接口为数据来源，实现订单准实时采集、脱敏清洗、SQLite 增量存储、网格化统计、热点预测、调度优化、GIS 可视化和简易 AI 助手。

## 运行方式

```bash
pip install -r requirements.txt
streamlit run app.py
```

浏览器访问：

```text
http://localhost:8501
```

SQLite 数据库会在程序启动时自动检测并创建：

```text
data/database/bike_hotspot.db
```

复制 `.env.example` 为 `.env`，填写开放平台接口凭据：


## 业务建模口径

梁山县共享电单车可能存在停车区域，共享单车也可能没有固定站点，但通常存在运营范围或管理区域约束。因此系统不做“真实站点级预测”，而是采用以下口径：

- 将订单起点、终点经纬度映射到规则网格中；
- 将网格区域视为“虚拟站点”；
- 对运营区域约束下的网格级热点进行预测；
- 基于经纬度网格完成短时借还需求预测和供需缺口计算。

对于存在停车区域的共享电单车场景，网格可作为停车区管理的补充分析单元；对于无固定站点的共享单车场景，网格可用于识别运营范围内的短时借还需求和停放压力。

## 预测模块

系统采用分层智能预测机制，前端页面只提供“生成预测结果”入口，不暴露复杂模型选择项。天气和节假日特征默认参与预测流程。

默认特征包括：

- 小时、星期、是否周末；
- 当前小时借车量和还车量；
- 最近 1 至 2 小时滞后特征；
- 3 小时滚动均值；
- 历史均值；
- 日均温、降水量、最大风速、是否节假日。

内部自动选择预测方式：

- 有效训练样本较少时，使用“历史均值 + 最近趋势修正”模型，保证系统可运行；
- 样本达到一定规模后，使用 `GradientBoostingRegressor` 分别预测借车需求和还车供给；
- 序列样本充足且本地安装 PyTorch 时，尝试启用 CNN-BiLSTM 模型，使用过去 6 小时序列预测下一小时借还需求；
- PyTorch 未安装或深度模型训练失败时，系统自动回退到轻量模型或历史趋势模型。

预测结果写入 `prediction_results`，核心字段包括：

- `predicted_start_count`：预测借车需求；
- `predicted_end_count`：预测还车供给；
- `demand_gap`：供需缺口，定义为 `predicted_start_count - predicted_end_count`；
- `risk_level`：按供需缺口划分的低、中、高缺车风险等级。

## 调度模块

调度模块基于 `prediction_results` 中的预测结果生成，不脱离预测结果单独计算。系统采用“基于供需缺口与空间距离约束的启发式调度算法”：

- `demand_gap > threshold` 的网格为缺车区；
- `demand_gap < -threshold` 的网格为余车区；
- 从余车区向缺车区调度车辆；
- 优先匹配缺口大、距离近的区域；
- 调度数量不超过余车区可调出量。

调度结果写入 `dispatch_plans`，并在页面展示调度前总缺口、建议调度车辆数、调度后剩余缺口、缺口缓解率、调度任务数、总调度距离、任务表和调度路线地图。

## 主要功能

- 数据采集：按时间范围调用开放接口，支持手动采集和最近三天快捷采集；
- 增量入库：以 `orderGuid` 去重，重复订单自动跳过；
- 隐私保护：手机号不入库、不展示，用户标识不保存原文；
- 数据清洗：过滤异常经纬度、无效距离和无效时长；
- 网格统计：按网格小时统计借车量和还车量；
- 热点监测：展示起点/终点热力图、热点排行榜和小时趋势；
- 热点预测：生成网格级短时借还需求、供需缺口和风险等级；
- 调度优化：根据预测缺口生成车辆调度建议；
- GIS 可视化：展示梁山县边界、热点、预测风险和调度路线；
- 天气节假日：采集气象数据并生成节假日日历，作为预测特征；
- AI 助手：仅基于聚合统计、预测结果和调度摘要生成解释，不发送原始订单明细；
- 用户管理：支持管理员、运营人员、普通用户三类角色。

## 论文可用描述

本系统采用本地 Web 化方式实现，基于 Streamlit 构建交互式界面，使用 SQLite 数据库存储共享单车订单数据。系统通过山东公共数据开放平台接口获取梁山县哈啰共享单车订单数据，在数据入库前对手机号和用户标识等敏感字段进行脱敏处理。系统支持订单数据的增量更新，能够随着接口持续调用不断积累历史样本。在此基础上，系统将共享单车订单中的起点、终点经纬度映射到规则网格中，将网格区域视为虚拟站点，完成运营区域约束下的网格级热点识别、基于经纬度网格的短时借还需求预测和车辆调度方案生成，并通过 GIS 地图进行可视化展示。该系统主要面向共享单车短时运维调度场景，可为县域共享单车停放治理和精细化管理提供辅助决策参考。
